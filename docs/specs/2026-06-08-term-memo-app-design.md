# 用語暗記アプリ — 設計メモ & UI/UX 引継ぎ書

- 作成日: 2026-06-08
- 対象: 個人用（航空運航支援業務の用語を覚えるための学習アプリ）
- このドキュメントの役割: 機能・データ・ロジックを確定し、**UI/UX は Claude Design が作れるように**まとめた引継ぎ書。これ単体で全体像が分かるように書いています。

---

## 0. 一行で言うと
**自分の iPhone だけで動く、用語の定義を一言一句覚えるための学習アプリ。** クイズ（×〇）ではなく「定義をそのまま確認」する方式。間隔復習で出題し、進捗を見られる。用語は自分で追加でき、補足も付けられる。

## 1. 確定した方針・制約（変更不可の前提）
これは UI/UX を作る上での土台。ここを崩す提案はしないこと。

| 項目 | 内容 |
|---|---|
| 動かす場所 | **iPhone の Safari** で開く Web アプリ。「ホーム画面に追加」してアプリのように起動（PWA）。 |
| 開発環境 | 作るのは Windows。Mac/Xcode は使わない（ネイティブ iOS アプリではない）。 |
| 保存場所 | **すべて端末内**（ブラウザの localStorage）。端末ごとに独立。サーバー・アカウント・ログインなし。 |
| 通信 | 基本オフラインで動く。通信が要るのは「AI に相談」を押した時だけ（任意機能）。 |
| コスト | 本体は無料。AI を使う場合のみ、利用者自身の AI 利用キーで少額課金。 |
| 利用者 | 本人 1 人。共有・公開・マルチユーザーは対象外。 |
| 優先順位 | **土台はあくまで「学習アプリ」。AI はあとから足せる“おまけ”。** AI なしで全機能が成立すること。 |

## 2. ゴール / 非ゴール（YAGNI）

### ゴール（やること）
- 用語の **定義を一言一句** 確認できる（読む辞書 + 復習）。
- 用語を **自分で追加・編集・削除** できる。補足も付けられる。
- **文字でも音声でも入力**できる（音声は iOS 標準のキーボード🎤に任せる。アプリ側で録音機能は作らない）。
- **間隔復習**（覚えるほど出題間隔が伸びる）で効率よく定着。
- **進捗が見られる**（定着率・苦手・カテゴリ別・連続日数・復習待ち数）。
- **バックアップ**（書き出し／読み込み）でデータを守れる。
- （任意）**AI が進捗と“間違い記録”を読んで弱点傾向とアドバイスを返す**。

### 非ゴール（やらないこと）
- ×〇／4択クイズ、採点・点数。
- アカウント／ログイン／クラウド同期／複数端末リアルタイム共有。
- SNS 共有、ランキング、他者との比較。
- ネイティブ iOS アプリ化、App Store 配布。
- 音声の録音・保存、読み上げ（TTS）。（将来検討の余地はあるが今回は対象外）

## 3. 画面構成（情報設計）

下部タブで 4 画面を切替（親指で届く位置に）。設定は歯車アイコンから。

```
┌─────────────────────────────┐
│            画面本体              │
│                              │
│                              │
├─────────────────────────────┤
│  学習 │ 一覧 │  ＋追加 │ 進捗    │  ← 下部タブ
└─────────────────────────────┘
            （右上に ⚙ 設定）
```

1. **学習** … 今日の復習を進める（アプリの主役）
2. **一覧（辞書）** … 全用語を読む・探す・編集する
3. **＋追加** … 新しい用語を登録
4. **進捗** … 学習状況を見る ＋「AI に相談」
5. **設定**（タブ外・歯車） … バックアップ、復習スタイル、テーマ、AI キー

## 4. 各画面の詳細（UI/UX 設計の中心）

### 4-1. 学習（復習）画面 ← ここが最重要
今日出すべき用語（後述の「復習待ち」）を 1 件ずつ順番に出す。

**復習スタイルは 2 つ。設定で既定を選び、学習画面で都度切替もできる。**

#### ① 読むだけモード（既定・軽い）
1. 用語（表）だけを大きく表示
2. タップ／「定義を見る」で **定義＋補足を全文表示**（隠さず一言一句）
3. 下部に評価ボタン3つ：**覚えてた／あいまい／忘れてた**
4. 押すと次に出す日が自動で決まり、次の用語へ

#### ② 入力モード（任意・AI 分析の材料がたまる）
1. 用語（表）を表示
2. 利用者が **定義を自分で書く／話す**（テキスト入力欄。🎤で音声可）
3. 「答え合わせ」で **正解の定義を表示**し、**自分が書けなかった／違った部分をハイライト**（→ §7）
4. 評価ボタン3つ（同上）
5. 解答内容と「抜けた部分」が端末に記録される（AI 分析の材料）

- **状態**: 復習待ちが 0 件のとき → 「今日の復習は完了！」のねぎらい表示（空状態）。新規追加した用語はすぐ復習待ちに入る。
- **大事な UX**: 定義は読む対象＝主役。**可読性最優先**（大きめ文字・広い行間・高コントラスト）。
- 評価ボタンは下部に大きく、親指で押しやすく。

### 4-2. 一覧（辞書）画面
- 登録した全用語をリスト表示（用語＋定義の冒頭）。
- **検索**（用語・定義・補足を対象）。
- **カテゴリで絞り込み**（カテゴリ未設定の用語もまとめて見られる）。
- 並び順: 50音／追加が新しい順／定着が低い順 などを切替。
- 行タップで詳細表示（用語・定義・補足・カテゴリ・覚え具合）→ そこから編集・削除。
- 各用語の「覚え具合」を小さく可視化（例: 定着バッジ／レベル点）。
- **空状態**: 用語ゼロのとき → 例付きで「最初の用語を追加しよう」へ誘導。

### 4-3. ＋追加 / 編集 画面
- 入力欄: **用語（必須）／定義（必須）／補足（任意）／カテゴリ（任意）／要点キーワード（任意）**。
  - 各欄は通常のテキスト入力。**音声入力は iOS キーボードの🎤で自動対応**（アプリ側に録音ボタンは作らない）。
  - 「要点キーワード」= 入力モードの採点用に「これは外せない要点」を任意で登録（例: NOTAM →「航空」「運航上重要」「通知」）。未設定でも動く（→ §7 のフォールバック）。
- カテゴリは過去入力からの候補サジェスト（自由入力＋既存から選択）。
- 保存時バリデーション: 用語・定義が空なら保存不可（やさしいエラー表示）。
- 編集画面は同じフォームを再利用。削除は確認つき。

### 4-4. 進捗画面 ← AI の置き場所
無料の数字（土台）と、任意の AI 相談（おまけ）の 2 段。

**無料で出す指標:**
- **定着率**（定着した用語の割合。定着 = レベル3以上、§6）
- **カテゴリ別の定着**（カテゴリごとの定着率バー）
- **苦手ランキング**（「忘れてた／あいまい」が多い順、上位10件程度）
- **連続学習日数**（streak）
- **今日の復習待ち数**

**任意の AI 相談:**
- 「**AI に相談**」ボタン。設定で AI キーが登録済みのときだけ有効。
- 押すと、進捗の数字と（入力モードの）間違い記録を読んで、AI が **用語ごと／カテゴリごとの弱点傾向＋次の一手** を日本語で返す（→ §8）。結果はパネル表示・コピー可。
- キー未登録なら、ボタンは「設定で AI を有効化すると使えます」と案内（押せるが設定へ誘導）。

### 4-5. 設定画面（歯車）
- **バックアップ**: 「書き出し」「読み込み」（→ §9）。
- **復習スタイルの既定**: 読むだけ／入力（→ 4-1）。
- **テーマ**: 端末に合わせる／ライト／ダーク。
- **AI（任意）**: 有効/無効トグル、AI 利用キーの入力（伏字）、プロバイダ／モデル選択。**キーは端末内のみ保存・書き出しには含めない**。「使うと自分の学習データが AI 提供元へ送られます」という注意書きを添える。

## 5. データモデル（実装の基準）

localStorage に 1 つの JSON で保存（キー例: `term-memo-app/data`）。AI キーだけは別キー（例: `term-memo-app/aiKey`）に保存し、**書き出しには含めない**。

```jsonc
// 用語 1 件
Term = {
  "id": "string",            // 一意ID
  "term": "string",          // 用語（必須）
  "definition": "string",    // 定義（必須・一言一句）
  "note": "string",          // 補足（任意：読み/英語/正式名称もここへ）
  "category": "string",      // カテゴリ（任意・空可）
  "keywords": ["string"],    // 要点キーワード（任意・入力モード採点用）

  // ↓ 進捗（自動管理。利用者は直接触らない）
  "level": 0,                // 0..6（覚え具合）
  "dueDate": "2026-06-08",   // 次に出す日（YYYY-MM-DD）。新規は当日。
  "lastReviewed": null,      // 最後に復習した日 or null
  "counts": { "good": 0, "hard": 0, "again": 0 },
  "attempts": [              // 入力モードの解答履歴（直近10件のみ保持）
    { "date": "2026-06-08", "text": "利用者の解答", "missed": ["抜けた要点"] }
  ],
  "createdAt": "2026-06-08T09:00:00Z",
  "updatedAt": "2026-06-08T09:00:00Z"
}

// アプリ全体
AppData = {
  "version": 1,
  "terms": [ /* Term[] */ ],
  "studyDates": ["2026-06-07", "2026-06-08"],  // 学習した日（連続日数の算出用）
  "settings": {
    "reviewMode": "read",      // "read" | "input"（既定 read）
    "theme": "system",          // "system" | "light" | "dark"
    "aiEnabled": false,
    "aiProvider": "",           // 例: "openai" | "anthropic" | "google"
    "aiModel": ""
    // aiApiKey はここに入れない（別保存・export除外）
  }
}
```

## 6. 間隔復習アルゴリズム（シンプル段階式）

- 各用語に `level`（0〜6）と `dueDate`。
- **レベル別の次回間隔（日）**: `{0:1, 1:3, 2:7, 3:14, 4:30, 5:60, 6:120}`
- 評価を押したときの処理（基準日 = 今日）:
  - **覚えてた**: `level = min(level+1, 6)` → `dueDate = 今日 + INTERVAL[level]`、`counts.good++`
  - **あいまい**: `level` 据え置き → `dueDate = 今日 + 1日`、`counts.hard++`
  - **忘れてた**: `level = 0` → `dueDate = 今日`（その場のセッションに再投入）、`counts.again++`
- **新規用語**: `level=0`, `dueDate=今日`（すぐ復習待ち）。
- **復習待ち（due）** = `dueDate <= 今日` の用語。並びは「期限切れが古い順 → 新規」。
- **定着** = `level >= 3`。`定着率 = 定着した用語数 / 総用語数`。
- ×〇判定はしない。スケジュールは自己申告だけで動く。

## 7. 「抜けた部分」のハイライト仕様（入力モード・無料）

入力モードで「答え合わせ」したとき、正解の定義のうち **利用者が書けなかった/違った部分**を色＋下線などで示す。実装は次の優先で:

1. **要点キーワードがある場合（推奨・精度高）**: その用語の `keywords` のうち、解答テキストに含まれないものを「抜けた要点」とし、正解定義中の該当語をハイライト。`missed` に記録。
2. **キーワードがない場合（フォールバック）**: 解答と正解定義の **文字単位の最長共通部分（LCS）** を取り、正解側で一致しなかった連続部分をハイライト。簡易なので完璧でなくてよい（自分用の目安）。

- 日本語は語の区切りが難しいため、形態素解析ライブラリは使わない（単一ファイル維持のため）。上記の素朴な方法で十分。
- ハイライトは **色だけに頼らない**（下線/マーカー併用）= 視認性・アクセシビリティのため。
- 結果（解答テキストと `missed`）を `attempts` に追記（直近10件で打ち切り）。これが AI 分析の材料になる。

## 8. AI 連携仕様（任意・差し込み口）

**位置づけ**: 本体とは完全に分離した“おまけ”。無効でもアプリは全機能成立。

- **起動**: 進捗画面の「AI に相談」を押した時だけ。自動・常時呼び出しはしない（コスト最小）。
- **AI に渡す内容（payload）**: トークンを抑えるため要約して渡す。
  - 全体サマリ（総数、定着率、レベル分布）
  - カテゴリ別の定着率
  - 苦手な用語 上位 ~15 件（用語・定義・`counts`・直近の `missed`/解答）
- **AI に求める出力**: 日本語で簡潔に。「**用語ごと／カテゴリごとの弱点傾向**」＋「**次にやるべき復習の一手**」。
- **呼び出し方**: 設定に保存した利用者自身の AI 利用キーで、ブラウザから AI 提供元の API を直接呼ぶ（サーバーを増やさない）。
- **実装上の注意（正直に）**:
  - 提供元によってはブラウザからの直接呼び出し（CORS）を制限する場合がある。実装時に対応プロバイダを確認すること。回避にプロキシを置くと「サーバーなし」が崩れるので、まずは直接呼べる構成で試す。
  - 使った分だけ少額課金（用語 100〜300 語規模の相談なら 1 回あたり数円〜十数円想定）。
  - **プライバシー**: この機能を使う瞬間だけ、本人の学習データが AI 提供元へ送られる。設定画面に明記する。
- **キー未登録時**: 機能は無効。設定への導線だけ見せる。

## 9. バックアップ仕様（書き出し／読み込み）

- **書き出し**: `AppData`（terms＋settings、**AI キーは除外**）を JSON ファイルとしてダウンロード。ファイル名例 `term-memo-backup-YYYYMMDD.json`。
- **読み込み**: ファイル選択（`<input type="file">`）→ 解析 → `version` 検証 → **全置き換え**（確認ダイアログ後）。マージは将来検討（今回は全置き換えで十分）。
- **iOS 補足**: 書き出しは「ファイル」アプリに保存され、共有シートからも扱える。読み込みはファイル選択で「ファイル」アプリ等から取得。標準の input/anchor で実現可能。
- 端末内で完結・サーバー不要。

## 10. PWA / オフライン

- `manifest.json`（アプリ名・アイコン・`display: standalone`・テーマカラー）と、iOS 用メタ（`apple-touch-icon`、`viewport`、`apple-mobile-web-app-capable`）を用意 → 「ホーム画面に追加」でアプリのように起動。
- データは localStorage なのでオフラインで動作。**アプリ本体をオフラインでも起動**したい場合は Service Worker でアプリシェルをキャッシュ（推奨だが任意）。

## 11. UI/UX ガイド（Claude Design 向け）

**ビジュアルの個性は Claude Design に委ねる。** ただし以下は外さないこと。

- **対象端末**: iPhone 縦持ち・Safari・片手操作。基準サイズ iPhone 14/15（390×844pt）想定、可変対応。
- **ナビ**: 下部タブ（学習／一覧／＋追加／進捗）＝親指で届く位置。設定は歯車。
- **可読性が最優先**: 定義テキストが主役。大きめの日本語フォント、広い行間（目安 line-height 1.7〜1.9）、十分なコントラスト、適切な 1 行字数。文字サイズ拡大（Dynamic Type）にも耐える。
- **ライト／ダーク両対応**（端末設定に追従）。学習は明暗いろいろな環境で行う。
- **タッチターゲット 44×44pt 以上**。復習の評価ボタンは下部に大きく。
- **復習カードが主役**: 1 画面 1 用語、余計な装飾を減らす。表→裏の表示は控えめなアニメで。
- **空状態を丁寧に**: 用語ゼロ／復習完了の画面を用意（例文・ねぎらい）。
- **マイクロコピーは日本語・落ち着いた実務トーン**（仕事の学習ツール）。
- **アクセシビリティ**: VoiceOver ラベル、色だけに意味を持たせない（ハイライトは下線併用）。
- **音声入力に専用 UI は不要**: 通常の入力欄でよい（iOS キーボードの🎤が担う）。録音ボタンを作らないこと。

## 12. 受け入れ基準（Definition of Done）

- [ ] 用語を 用語/定義/補足/カテゴリ/要点キーワード で追加・編集・削除できる。
- [ ] オフラインで動作し、アプリ再起動後もデータが残る（localStorage）。
- [ ] ホーム画面に追加でき、単体アプリのように起動する（PWA）。
- [ ] 復習で「復習待ち」が出題され、評価で §6 の通りスケジュールが更新される。
- [ ] 「読むだけ」「入力」両モードが動作。入力モードは「抜けた部分」をハイライトし記録する。
- [ ] 進捗画面に 5 指標（定着率・カテゴリ別・苦手ランキング・連続日数・復習待ち数）が出る。
- [ ] バックアップの書き出し→読み込みでデータが復元できる。
- [ ] AI: キー登録時は弱点アドバイスが返る／未登録でも本体は完全に動く。
- [ ] 成果物は **単一の自己完結した静的物**（1 つの HTML、または小さな静的一式）で、Safari で開く／静的配信で動く。

## 13. Claude Design への引継ぎメモ
- まず作るのは **UI/UX と画面遷移**。本書の §3〜§4・§11 が中心。ロジック（§5〜§9）はこの通り実装すれば破綻しない。
- **本体（AI なし）だけで全機能が成立する**ように。AI は §8 の「差し込み口」を用意するだけでよい（最初は無効で OK）。
- **委ねてよい点（決め打ちでよい）**: アプリ名、配色・トーン・アイコン、進捗グラフの見せ方、アニメの度合い、フォント選定、オフライン用 Service Worker を入れるか。
- **崩してはいけない点**: §1 の前提（iPhone Safari / 端末内保存 / 無料・サーバーなし / 単一静的物 / 学習アプリが主・AI はおまけ）、×〇クイズにしない、音声は iOS 標準に任せる。
- 不明点が出たら、この設計メモの該当章番号を添えて確認すること。
